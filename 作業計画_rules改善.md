# ルールファイル（rules.md）改善計画書

## 1. 目的
`edit-rules.md` の内容を整理・構造化し、AI（Antigravity）がすべてのルールをより確実に理解し、遵守できるように最適化します。

## 2. 改善方針
- **マークダウン構造の最適化**: 適切な H1, H2, H3 タグを使用し、AIが階層構造を把握しやすくします。
- **重複内容の整理**: 「Dead Code Cleanup」などの重複箇所を統合し、簡潔かつ強力な指示にまとめます。
- **構成の再配置**: 
    1. 前提・基本方針
    2. クイックリファレンス（最重要項目）
    3. アーキテクチャ（4層構造）
    4. コーディング・型安全ルール
    5. ステート・サイドエフェクト管理
    6. スタイリング・UI
    7. ディレクトリ構造
    8. ワークフロー・AIツール最適化
    という論理的な順序で再配置します。
- **指示の明確化**: 曖昧な表現を避け、AIが「何をしてはいけないか」「何をすべきか」を即座に判断できる命令形・ルールベースの記述にリライトします。
- **ディレクトリ構造の維持**: 指示通り、既存のディレクトリ構造や絶対パスのルールなどは変更しません。
- **日本語ベース**: 基本的に日本語で記述します（技術用語は英語併記）。

## 3. 実施手順
1.  `edit-rules.md` の内容を読み込み、要素を抽出。
2.  上記の改善方針に基づき、マークダウン形式でリライト。
3.  リライトした内容で `edit-rules.md` を上書き（または要望通り `rules.md` としても機能する形に調整）。

## 4. 完了条件
- すべての既存ルールが含まれていること。
- マークダウンの見出しが適切であること。
- 重複が排除されていること。
- AIにとって可読性が高い（パースしやすい）構造になっていること。
