【Webアプリの構造】

★UIはReactのコンポーネントを組み合わせる
→HTML要素も関数中でreturn、スタイリングもTailwindのクラスを直接書くor shadcn/uiのスタイリング内蔵部品を呼び出す
→.tsxファイルのコンポーネントにHTML要素、スタイリング、機能がまとまった状態
(本体のHTMLは｢空っぽの器｣で、サイトのタイトル、アイコン、フォント読み込みくらいの要素しか持たない)

React：Stateに応じた描画を宣言、stateと常に同期し、stateの変化に応じて一斉に関数走らせ（計算）、一括でDOM操作（描画）→その後副作用
（state変化による仮想domでの計算と、dom操作や外部とのやり取り（副作用）は独立している）
（Vanilla TSは、それぞれの関数がそれぞれのタイミングで描画、関数同士の依存も多い）

※Next.jsでは、そもそも.htmlを触らない！


・Tailwind CSS: CSSの設定が、あらかじめ短いクラス名として登録されている辞書。CSSファイルを作るのではなく、Reactの中に直接タグを書き込む。
（Prettier + Tailwind Plugin・**Tailwind CSS IntelliSense**を使う！）

・shadcn/ui: radix-uiというライブラリでTailwindベースで作られた「ボタン」「入力フォーム」「メニュー」などのスタイリング済み機能


【Reactにない機能の実装】

・.tsファイルで独自ロジック組むor外部ライブラリ探す
→ラッパーし、カスタムフックでReactと連携
(外部ライブラリや独自ロジックは、ラッパーし、すべてReactに結びつき、直接DOM操作せず、React計算後の副作用のタイミングに合わせて動かすような仕組みにする！)
→イベントでstate遷移→計算→副作用として外部ロジック起動→state遷移→計算という2段階！
→いろいろなテクで隠す！！
（起動タイミング、破棄、範囲の制御をしっかり行う！）

（Reactライブラリは自動同期！）

(例)
Zustand（状態管理）
TanStack Query
Zod
