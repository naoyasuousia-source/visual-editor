# 目的

- 現在、出力HTMLは、本文が空の状態でも、700行超えと、非常に長い。
- AIに毎回読み込ませて利用するので、コンテキスト消費が大きいことは問題である。
- そこで、出力用HTMLを軽量化したい。

# 要件
- 自動編集後のハイライト機能など、出力HTMLに関係のない記述は、content.cssから削除し、Reactコンポーネントに記述する。
- content.cssは出力HTMLのスタイリングは完全に維持したまま、できるだけコンテキストを小さくする。
- AI向けのガイド文も、AIが利用方法を間違えないようなAIにとってわかりやすい記述は維持しながら、できるだけコンテキストを小さくする。

# 注意点
- 出力HTMLおよび、エディタのスタイリング、各機能のスタイリングは絶対に崩さないように注意する。
